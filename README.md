## 📌 О проекте

**Erudito** — полнофункциональная образовательная платформа в духе Udemy/Stepik, разработанная с нуля. Система поддерживает две роли пользователей с принципиально разными интерфейсами: **преподаватели** создают и управляют курсами через AJAX-редактор, **студенты** записываются на курсы и проходят уроки с отслеживанием прогресса.

Проект развёрнут на VPS и работает в продакшн-режиме (`DEBUG=False`), статика раздаётся через WhiteNoise без необходимости настройки отдельного Nginx для статических файлов.

## ⚙️ Архитектура системы

Erudito построен по классической **MVT-архитектуре** Django с дополнительным акцентом на разделение ответственности между ролями пользователей:

```
Браузер
   │
   ▼
┌─────────────────────────────────────────────┐
│               Django Router                 │
│  /           /courses/   /student/  /teacher│
└──────────────────────┬──────────────────────┘
                       │
          ┌────────────┴────────────┐
          ▼                         ▼
   ┌─────────────┐          ┌──────────────┐
   │   Views     │          │  Django Admin│
   │ (бизнес-    │          │  (Jazzmin)   │
   │  логика)    │          └──────────────┘
   └──────┬──────┘
          │
   ┌──────┴──────┐
   │  ORM Models │   User / Course / Lesson /
   │  (SQLite)   │   Enrollment / LessonProgress
   └─────────────┘
```

Взаимодействие интерфейса преподавателя с сервером реализовано через **AJAX + JSON API** без перезагрузки страниц, тогда как студенческий интерфейс использует стандартные Django-шаблоны с точечными AJAX-вызовами для прогресса.

## 👤 Ролевая система

Проект использует **кастомную модель пользователя** (`AbstractUser`) с заменой `username` на `email` в качестве идентификатора. Реализованы две роли:

| Роль | Доступные разделы |
|---|---|
| **Студент** | Каталог, страница курса, прохождение уроков, личный кабинет, отзывы |
| **Преподаватель** | Дашборд преподавателя, создание/редактирование/удаление курсов, управление уроками |

Доступ к страницам защищён декоратором `@login_required` с проверкой роли внутри view-функции. Попытка перейти на страницу другой роли приводит к редиректу.

## 🗄️ Модели данных

Семь взаимосвязанных моделей покрывают весь учебный процесс:

**`User`** — кастомная модель пользователя:

| Поле | Описание |
|---|---|
| `email` | Уникальный идентификатор (вместо username) |
| `role` | `student` или `teacher` |
| `first_name`, `last_name` | Обязательные поля при регистрации |

**`Course`** — курс:

| Поле | Описание |
|---|---|
| `title`, `description` | Название и описание |
| `category` | `dev` (Разработка), `design` (Дизайн), `data` (Аналитика) |
| `level` | `beginner`, `intermediate`, `advanced` |
| `teacher` | FK → User (только с ролью teacher) |
| `thumbnail` | Обложка (ImageField) |
| `is_published` | Флаг публикации — управляет видимостью в каталоге |

**`Introduction`** — введение в курс (OneToOne → Course): заголовок, текст и опциональная ссылка на видео.

**`Lesson`** — урок курса: порядок (`order`), длительность, видео-ссылка, текстовый материал.

**`LessonMaterial`** — прикреплённый материал к уроку: тип (`pdf`, `link`, `file`), файл или URL.

**`Enrollment`** — запись студента на курс. Содержит вычисляемые поля:
- `progress` — процент прохождения (0–100), вычисляется на лету из `LessonProgress`
- `lessons_done` — количество пройденных уроков
- `check_completion()` — автоматически помечает курс завершённым при 100% прогрессе
- `mark_accessed()` — обновляет время последнего визита

**`LessonProgress`** — прогресс по конкретному уроку. Метод `mark_done()` помечает урок пройденным и цепочкой вызывает `check_completion()` у связанного `Enrollment`.

**`Review`** — отзыв студента с оценкой 1–5. Доступен только после полного прохождения курса. Уникальная пара `(student, course)` гарантирует один отзыв на курс.

## 🎛️ Административная панель

Кастомная Django Admin с темой **Jazzmin (Darkly)** — полноценный инструмент управления платформой, а не стандартная заглушка. Реализованы:

**Кастомные display-поля** для каждой модели: цветные бейджи ролей/категорий/уровней, прогресс-бары прохождения, аватары с инициалами, рейтинговые звёзды, встроенные превью видео-ссылок.

**Bulk-actions** (групповые действия):
- Активировать / заблокировать пользователей
- Опубликовать / снять с публикации курсы
- Отметить записи завершёнными или сбросить прогресс
- Отметить уроки пройденными / сбросить прохождение

**Nested inlines** (`django-nested-admin`): редактирование Курс → Введение → Уроки → Материалы в одной форме.

**Оптимизация запросов**: все Admin-queryset используют `select_related`, `prefetch_related` и `annotate` (Count, Avg, Sum) для минимизации числа SQL-запросов при загрузке списков.

**Умные ссылки**: каждый объект в списке содержит кликабельные ссылки на связанные объекты (студент → профиль, курс → редактирование, урок → карточка урока).

## 💡 Ключевые технические решения

**AJAX-редактор курса.** Весь интерфейс создания и редактирования уроков работает без перезагрузки страницы: добавление/удаление/редактирование уроков и материалов отправляет JSON-запросы на `/teacher/...` эндпоинты, ответы обновляют DOM. Drag-and-drop сортировка уроков сериализует новый порядок и отправляет его одним запросом.

**Умная навигация по урокам.** При попытке студента перейти к уроку, находящемуся дальше первого непройденного, система делает редирект на актуальный урок. Это предотвращает пропуск контента и реализовано на уровне view без JavaScript.

**Вычисляемый прогресс без лишних запросов.** `Enrollment.progress` вычисляется динамически через `LessonProgress`, без хранения процента в БД — это гарантирует актуальность данных. В Admin для списков прогресс аннотируется через `Count` в одном запросе.

**Embed-конвертер видео.** `_to_embed_url()` автоматически преобразует обычные ссылки YouTube и Vimeo в embed-формат для iframe, используя регулярные выражения. Для YouTube используется `youtube-nocookie.com` с отключёнными рекомендациями.

**Оптимизированный каталог.** Главная страница выбирает топ-2 курса из каждой из трёх категорий отдельными annotate-запросами, затем объединяет и сортирует их в Python — это позволяет избежать сложного UNION-запроса и сохраняет читаемость кода.

**Защита от скипа в отзывах.** Эндпоинт `submit_review` проверяет `enrollment.is_completed` перед сохранением — студент не может оставить отзыв, не пройдя курс полностью. Модель использует `unique_together = ('student', 'course')` для гарантии одного отзыва на курс на уровне БД.

**Кастомный пользователь с email-авторизацией.** Модель `User` расширяет `AbstractUser` с заменой поля `username` на `email`, переопределением `USERNAME_FIELD` и кастомным `UserManager`. Это требовало явного указания `AUTH_USER_MODEL` в settings и использования `limit_choices_to` в Foreign Key курса для фильтрации только преподавателей.
